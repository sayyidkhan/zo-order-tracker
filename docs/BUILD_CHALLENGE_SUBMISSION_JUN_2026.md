## One Sentence About It

ZOrder is a lightweight storefront and order tracker that lets small sellers and businesses publish products, collect payment proof, and manage customer orders from one simple workspace.

We also built one-push deployment, so it is easy to get started selling products today.

Source code: https://github.com/sayyidkhan/zo-order-tracker

## Purpose For Each Route

- `/user` lets customers browse products, upload payment proof, and track orders.
- `/admin` lets sellers manage orders, inventory, branding, payments, and workflow rules.
- `/intro` explains the product.
- `/tech-stack` shows the architecture.
- `/why-zo-computer` explains how Zo helped ship and deploy it quickly.

## How Did You Use Zo?

I used Zo Space to build and iterate the project, Zo Sites to publish the live app, and Zo Services/Agents to support the order-tracking workflow, backend logic, and automation around storefront orders.

I also used Zo skills for deployment and syncing code between GitHub and Zo Computer.

Zo can be used to view and update inventory through channels such as Zo SMS or Telegram without requiring the seller to understand or manage infrastructure. This is the key operator benefit: small sellers can focus on orders, products, and customers instead of cloud setup, deployment, and backend operations.
